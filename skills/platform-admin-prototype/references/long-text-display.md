# 超长业务文本展示规则

本文件只约束 PC 后台原型中的业务文本和数据展示。它不约束产品注释内容，也不要求修改官方组件已经提供的溢出行为。

## 核心原则

- 数据层始终保留完整原始值。搜索、复制、编辑、提交和业务处理都使用完整值，禁止先截断再保存或传递。
- 根据实际容器宽度判断是否溢出，禁止用固定字符数切割字符串后拼接省略号。
- 只有内容真实溢出时才显示 `…`，也只有真实溢出时才提供显示完整原始值的 Tooltip。
- 紧凑展示区域可以截断；正常详情正文优先完整换行展示。
- 正在编辑的输入框、文本域和其他可编辑控件不得截断用户当前输入。
- 优先使用本地平台组件和 Arco 已有能力。确实缺少统一能力时，才增加本地共享封装；禁止为每个页面复制一套省略 CSS。

## 按任务重要程度分级

长文本不能统一使用固定行数或 Tooltip。先判断它对用户当前任务的重要程度，再选择展示方式：

| 信息级别 | 默认展示方式 | 典型字段 |
| --- | --- | --- |
| 当前决策必需信息 | 默认完整展示，不折叠 | 当前驳回/拒绝理由、当前整改要求、当前补充材料要求、错误或失败原因、风险与安全警告、危险操作影响说明、当前下一步处理说明 |
| 历史长文本业务记录 | 默认最多三行；真实超出后提供原位置“展开全文 / 收起” | 历史驳回理由、历史审批意见、处理备注、跟进记录、关闭/取消/终止原因、异常/故障/问题描述、投诉/反馈内容、申请/变更说明 |
| 快速识别信息 | 标题等标识字段默认单行；短摘要可最多两行；真实溢出时 Tooltip | 标题、名称、人员名称、部门名称、文件名称、树节点名称、紧凑卡片标题和短摘要 |

同一个字段在不同任务场景中可以使用不同策略：

- 当前正在处理的驳回理由或整改要求属于决策必需信息，应完整展示。
- 历史记录中的同类内容属于历史长文本业务记录，默认三行并支持展开/收起。
- 列表中用于快速识别该记录的标题仍使用单行省略和 Tooltip。

长段正文禁止使用 Tooltip 承载完整内容。Tooltip 只用于单行或短摘要等快速识别信息；需要连续阅读的长文本必须在页面原位置展开，保留原始换行和段落。

## 两类长度限制必须分离

每个可能超长的字段都需要分别判断以下两类限制，禁止混为一谈：

- **数据长度限制**：允许输入、保存或接口接收的最大字符数，属于字段定义和表单校验规则。优先使用 PRD 明确规定的限制；PRD 未定义时，不得为了页面展示效果擅自截断保存数据。
- **展示长度限制**：字段在特定页面区域中显示为单行、指定行数或完整换行，属于布局和展示规则。展示限制由实际容器宽度和最大展示行数决定，不按数据最大字符数决定。

页面发生省略时，数据层仍保留完整值；输入框允许输入多少字符，也不能反向决定列表、详情头部或卡片中固定显示多少字符。

## 复合文本宽度分配

当一个展示区域由多个字段组合而成，例如 `业务编号 · 业务标题`、`人员名称 + 部门` 或 `头像 + 主文本 + 副文本`，必须先拆分内部角色，再决定省略位置：

- **固定字段**：业务编号、状态、图标、头像、分隔符和操作按钮等。默认完整展示并禁止收缩。
- **弹性字段**：标题、名称、描述等可能超长的业务文本。占用固定字段之外的剩余宽度，并允许在真实溢出后省略。
- 只对真实溢出的弹性字段显示完整值 Tooltip，不能对整个组合字符串统一截断或统一显示 Tooltip。
- 固定字段、图标、头像和操作区不得被弹性文本挤压。

详情页紧凑头部中的 `业务编号 · 业务标题` 默认按以下规则处理：

- 整体最多显示一行。
- 业务编号和分隔符完整展示、不收缩。
- 业务标题使用剩余可用宽度，实际溢出后显示 `…`。
- 仅当业务标题真实溢出时，悬浮标题显示完整标题。
- 正常详情内容区仍应提供完整标题或完整业务内容。

## 场景决策

| 场景 | 默认展示规则 | 完整值访问方式 | 实现要求 |
| --- | --- | --- | --- |
| 普通表格文本列 | 单行省略 | 真实溢出时 Tooltip | 为对应列显式配置 Table 的 `ellipsis` 与 `tooltip` 能力 |
| 头像 + 主副文本等复合表格单元格 | 主文本、副文本分别单行省略 | 各自真实溢出时显示完整值 | 文本容器必须允许收缩；头像、图标和操作区不得被挤压 |
| 详情页紧凑复合标题 | 固定字段完整展示，弹性标题单行省略 | 弹性标题真实溢出时 Tooltip | 编号、分隔符等固定字段不收缩；标题使用剩余宽度 |
| 普通列表标题 | 单行省略 | 真实溢出时 Tooltip | 使用 Typography 或统一文本展示封装 |
| 短列表摘要、卡片摘要、业务通知摘要 | 默认最多两行省略 | 真实溢出时 Tooltip；有详情入口时可进入详情查看 | 仅适用于快速浏览的短摘要，不截断关键操作和状态 |
| 紧凑详情摘要字段 | 单行省略 | 真实溢出时 Tooltip | 普通详情内容区不使用该规则，优先换行完整展示 |
| 树节点名称 | 单行省略 | 真实溢出时 Tooltip | 使用节点插槽配合 Typography/Tooltip；展开图标和节点操作不得被挤压 |

## 默认不得截断

以下内容默认完整展示或通过调整列宽、布局、换行和复制能力保证可用，不得直接套用省略规则：

- 状态、金额、数量、日期和时间。
- 关键标识符，例如工单号、订单号和业务编号。空间确实不足时，优先调整布局并提供复制能力。
- 操作按钮名称。
- 错误信息、必填校验信息和其他需要用户立即理解的反馈。
- 上文定义的当前决策必需信息。
- 正在编辑的输入内容。
- 正常详情页的重要正文内容。

## 官方组件边界

以下组件优先使用其官方或本地封装已有能力，不额外增加通用截断规则：

- Select / Dropdown：使用已有的选中项、选项省略、`max-tag-count`、`tag-nowrap` 等能力。
- Tag / 状态标签：使用组件自身的紧凑与省略能力。
- Menu、Breadcrumb、Tabs：使用组件自身的折叠、最大数量、滚动等能力。
- Upload 文件名：使用组件已有的文件名省略能力。
- Notification：完整换行展示通知内容，不截断。
- 图表、按钮：不在本规则中处理。

Tree 节点名称不属于上述豁免范围。Arco Tree 没有完整的节点标题省略加悬浮全文能力，需要在节点插槽内显式实现。

## 实现顺序

1. 表格普通列优先使用 Table 列级 `ellipsis + tooltip` 配置。
2. 复合文本先拆分固定字段和弹性字段。固定字段禁止收缩，弹性字段设置可收缩的剩余宽度后再处理实际溢出。
3. 单行或短摘要优先使用本地平台组件或 Arco Typography 的 `ellipsis`、`rows` 和 `showTooltip` 能力。
4. 历史长文本业务记录使用最多三行的可展开文本，真实超出后显示“展开全文 / 收起”；当前决策必需信息直接完整展示。
5. 只有平台组件无法覆盖且多个页面确实复用时，才创建共享的长文本展示封装。
6. 自定义 Tooltip 或“展开全文”必须通过实际溢出检测控制显示，不能让未溢出的普通文本出现多余交互。

## 验证数据

实现和验收时至少覆盖：

- 超长中文标题或名称。
- 无空格的长英文、邮箱或 URL。
- 主副文本同时很长的复合单元格。
- `业务编号 · 超长业务标题` 等固定字段与弹性字段组合。
- 当前驳回理由、历史驳回理由和列表识别标题三种不同任务场景。
- 少于三行与超过三行的历史长文本记录，以及展开和收起后的内容完整性。
- 浏览器宽度或容器宽度变化后的表现。
- 数据长度校验与展示省略互不影响。
- 复制、编辑、搜索和提交仍使用完整原始值。
