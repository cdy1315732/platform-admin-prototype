# 图表需求规则

本文件在 PRD、需求、页面名称、模块意图或已有原型中出现图表、趋势、分布、占比、排行、统计可视化、指标看板、工作台分析、数据驾驶舱等图表类需求时读取。

如果模块需要用折线、柱状、饼图、环图、面积、散点、热力、漏斗、仪表盘、雷达、地图、关系图、桑基图或类似非卡片形式表达数据，就视为真实图表需求。没有图表需求时不要读取、不要访问 ECharts、不要为了展示而额外添加图表。

## 来源

图表选型参考 Apache ECharts 官方示例：

- https://echarts.apache.org/examples/zh/index.html

使用该站点的目的只是选择合适的图表表达和交互范式，不需要复制全部示例。

## 强制规则

- 真实图表模块必须先去 Apache ECharts 官方示例站点选择相近示例，再实现到本地 PC 后台页面。
- 不允许用手写 SVG、CSS-only 图表、div 柱形、canvas 手绘、手算 polyline、静态图片或伪图表替代真实图表，除非用户明确要求“只做静态占位”。
- 找到示例后，保留示例的图表类型、option 结构、tooltip、legend、axis、series 和基础交互思路；不要照搬示例的页面外观。
- 图表配色必须二次适配到本地产品视觉：平台蓝作为主色，搭配平台成功/警示/危险色或低饱和辅助色；背景保持白底卡片；文字、网格线、图例、tooltip 与本地后台风格一致。
- 如果无法访问 ECharts 示例站点，先说明阻塞原因；不要静默改用手写图表。只有用户确认接受临时方案时，才可以做清楚标注的临时占位。
- 实现后在代码或计划中保留可追溯信息：使用的 ECharts 示例名称或 URL、选择原因、已调整的本地配色规则。

## 简要图表分类

用于快速判断 PRD 里的图表需求是否有合适方向；实现时仍需去 ECharts 示例站点选择具体示例。

| 分类 | 常见图表 | 适用场景 |
| --- | --- | --- |
| 趋势变化 | 折线图、柱状图、面积图 | 时间趋势、增长变化、日/周/月统计。 |
| 对比排行 | 柱状图、条形图、象形柱图 | 排名、分组对比、Top N。 |
| 占比构成 | 饼图、环图、矩形树图、旭日图 | 分类占比、层级占比、构成分析。 |
| 关系流向 | 桑基图、关系图、线图 | 流量去向、节点关系、路径关联。 |
| 分布密度 | 散点图、涟漪散点图、热力图、箱线图 | 分布、异常点、密度、区间统计。 |
| 指标仪表 | 仪表盘、漏斗图、进度类图表 | 完成率、转化漏斗、单指标状态。 |
| 多维能力 | 雷达图、平行坐标 | 能力评分、多指标对比、画像分析。 |
| 地理空间 | 地图、地理坐标、线图 | 地域分布、区域统计、地理路径。 |
| 层级结构 | 树图、矩形树图、旭日图 | 分类层级、组织结构、层级数据占比。 |
| 金融/特殊 | K 线图、自定义系列 | 金融行情或 PRD 明确要求的特殊图形。 |

如果只是普通数字指标，优先使用指标卡，不要强行使用图表。

## 使用规则

1. 先判断 PRD 是否真的需要图表。普通指标卡、表格、详情字段不算图表需求。
2. 确认需要图表后，打开 ECharts 示例站点，根据数据意图选择相近示例；不要从记忆中手写一个大概样子。
3. 记录选中的示例名称或 URL、选择理由、图表类型、数据维度、tooltip/legend/axis/series 方案。
4. 图表放在本地后台页面结构中，通常承载在普通内容卡片或工作台模块里，不单独做成展示页。
5. 图表视觉必须服从本地后台规范：白底、平台蓝为主色、字号克制、卡片间距一致、无暗黑模式。
6. 基于示例 option 调整本地配色：主系列优先 `#165DFF`，辅助系列可用 `#00B42A`、`#FF7D00`、`#F53F3F`、`#722ED1`、`#86909C`，网格线用浅灰，坐标轴文字用弱文本色。
7. 图表需要具备基础交互：tooltip、legend、hover、必要的筛选联动或时间范围联动。
8. 图表容器要有稳定高度和响应式宽度，避免挤压、溢出或大面积空白。
9. 数据不足时使用合理样例数据；如果 PRD 给了指标口径，以 PRD 口径为准。
10. 如果本地项目尚未安装 ECharts，只在实现图表页面时添加依赖；不要因为没有图表需求而预装。

## 常见后台使用场景

- 工作台总览中的趋势、分布、占比、排行。
- 列表页上方的数据分析模块。
- 详情页中的历史趋势或处理过程统计。
- 设置或运营页面中的效果监控。

这些场景出现时再进入 ECharts 选型流程；否则继续使用本地组件体系完成页面。

## 失败模式

出现以下情况时，review 必须判为问题并修复：

- 页面里有“趋势”“分布”“占比”“排行”等真实图表模块，但代码没有 ECharts 依赖或没有 ECharts option。
- 图表用 `<svg>` / `<polyline>` / `<canvas>` / CSS 宽度条手写出来。
- 图表没有 tooltip、legend、hover 或必要的筛选联动。
- 图表直接套用 ECharts 示例默认配色，和本地后台页面不一致。
- 图表只是为了填充页面而出现，PRD/需求没有真实统计口径或分析需求。
