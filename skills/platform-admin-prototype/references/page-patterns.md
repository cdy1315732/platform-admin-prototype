# PC 后台页面模式

生成原型时先判断页面类型，再选择组件组合。不要把所有模块堆在一个页面上。

## 应用壳

默认后台结构：

- 左侧导航：只放一级业务页面。
- 顶部栏：面包屑、搜索、通知、用户信息。
- 内容区：当前页面标题和页面模块。

导航规则：

- 工作台、列表、分类管理、人员管理、系统设置等可以作为一级页面。
- 创建页、详情页、编辑页如果由按钮或表格行触发，属于二级页面，不放左侧导航。
- 进入二级页面时，左侧导航高亮它的父级页面。
- 面包屑只保留必要层级，例如 `工作台 / 数据列表 / 详情页`。

## 工作台 / 仪表盘

适用场景：

- 总览、待办、指标、快捷入口、近期操作。

规则：

- 指标卡片高度紧凑，文案到卡片上下间距一致。
- 左右栏模块之间使用统一间距，不因为网格行高产生大空白。
- 快捷入口是操作入口，不要做成营销模块。

## 列表页

适用场景：

- 查询、筛选、分页、批量操作、行操作。

推荐组成：

1. 页面标题。
2. `PlatformFilterPanel`。
3. `PlatformTableCard`。
4. 表格行操作、分页、批量操作。

规则：

- 导出/创建按钮放在表格卡片标题行右侧。
- 筛选区不要有重复标题和重复重置入口。
- 表格时间列可排序，默认不选中。
- 勾选后才显示批量操作。

## 创建 / 编辑表单页

适用场景：

- 从列表页点击创建、编辑进入。

推荐组成：

1. 页面标题或面包屑。
2. `PlatformFormCard`。
3. 表单字段。
4. 底部取消/提交按钮。

规则：

- 表单页属于二级页面时不放左侧导航。
- 页面级 `PlatformFormCard` 在内容区内水平居中，默认最大宽度 `960px`，窄内容区使用全部可用宽度且不产生横向滚动。
- 表单卡片内部标题、说明和字段左对齐，底部操作按钮右对齐。
- 标题和表单字段在同一张白色卡片中。
- 上传使用普通灰色按钮，默认无文件。
- 不要使用上传大拖拽蓝色区域作为默认方案。

## 详情页

适用场景：

- 单条数据完整信息查看、状态流转、操作记录、关联数据。

推荐组成：

1. `PlatformDetailPageTemplate`。
2. 详情头部：标题、辅助信息、操作按钮。
3. `PlatformDetailSummary`。
4. `PlatformSteps` 或流程节点。
5. 信息区、说明区、时间线。

规则：

- 详情页属于二级页面，不放左侧导航。
- 状态用圆点 + 文案。
- 流程使用流程节点，不用普通横向标签条。
- 外边框不要过多，避免层次复杂。

## 设置页

适用场景：

- 基础配置、通知配置、规则配置。

推荐组成：

- 左侧分组菜单。
- 右侧配置内容卡片。
- 底部保存/恢复默认按钮。

规则：

- 说明文案精简，重复的辅助文案要去掉。
- 表单模块与标题左对齐。
- 开关行与文字对齐一致。

## 分类 / 树管理页

适用场景：

- 分类树、组织树、菜单管理、权限树。

规则：

- 不拆成多个重复卡片；标题和树内容可以合并在一张主卡片内。
- 每个节点支持添加子级、编辑、删除、启用禁用。
- 开关必须使用正确样式，圆点居中。
- 节点操作靠右，节点层级通过缩进表达。

## 人员 / 权限管理页

适用场景：

- 人员列表、角色分配、启用禁用、当前负载。

规则：

- 表格标题使用“人员列表”等准确名称。
- 添加人员放在表格卡片标题行右侧。
- 筛选下拉必须有“全部角色”“全部状态”等默认项。

## 结果反馈

适用场景：

- PRD 明确要求提交后进入结果页。
- 弹窗内反馈操作结果。

规则：

- 默认不要为每个提交动作创建独立结果页。
- 如果使用结果页，必须使用本地 `PlatformResult` / `PlatformResultPageTemplate` 样式。
