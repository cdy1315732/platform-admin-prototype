# 图标与 Emoji 使用规则

本文件统一约束 PC 后台原型中的功能图标、导航图标、状态图标和装饰图标。

## 核心规则

- 禁止使用 Emoji、Unicode 符号或普通文本字符充当 UI 功能图标、导航图标、状态图标或按钮图标。
- 需要图标时，优先使用 `@platform/components` 已封装的图标或带图标组件。
- 本地平台没有对应图标时，使用 Arco Design Vue 官方 Icon。
- 禁止自行绘制 SVG、CSS 图形或使用来源不明的图片替代常规 UI 图标。
- 纯文本操作不强制添加图标。图标必须帮助识别功能，不能只为装饰而滥用。
- 图标尺寸、颜色和对齐遵循本地组件样式，不用 Emoji 或字符宽度修补布局。

禁止示例：

```text
🔍 搜索
⚙️ 设置
➕ 添加
❌ 删除
```

允许方式：

```vue
<IconSearch />
<IconSettings />
<IconPlus />
<IconDelete />
```

## 允许保留 Emoji 的情况

- 用户输入、接口数据或已有业务内容本身包含 Emoji 时，作为原始业务数据正常展示。
- PRD 明确要求 Emoji 是业务字段值、内容或产品表达的一部分时，可以保留。
- 上述例外不允许把 Emoji 重新用作按钮、导航、状态或功能图标。

## 图标来源优先级

1. `@platform/components` 已封装图标或组件自带图标。
2. Arco Design Vue 官方 Icon。
3. 仅在 PRD 明确要求品牌图形或专有业务图形时使用提供的正式资产。

若没有合适图标，优先使用清晰的纯文本操作，不得用 Emoji、Unicode 符号或临时自绘图标替代。

## 验证

- 全页面检查按钮、导航、状态、空状态、搜索、添加、删除、设置和上传等位置。
- 确认没有 Emoji、Unicode 符号、普通字符或临时自绘图形模拟 UI 图标。
- 确认图标来自平台组件或 Arco 官方 Icon，且不存在无意义的装饰性图标。
- 确认业务数据中的 Emoji 未被错误删除或替换。
