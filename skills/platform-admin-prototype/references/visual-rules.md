# 视觉与布局规则

本文件记录用户偏好的本地后台视觉规范。生成原型时以这些规则为准。

## 范围

- 只做 PC 端后台 / B 端 / Admin 原型。
- 不做移动端。
- 不做暗黑模式。
- 不做营销页、落地页、宣传页。
- 第一屏必须是实际可操作后台界面。

## 布局

- 真实后台系统优先：左侧导航、顶部栏、内容区。
- 不把所有模块堆在一个页面上。
- 页面通过导航、按钮、表格行、弹窗、抽屉形成真实工作流。
- 页面内容保持工作台密度，避免过度装饰。

## 卡片

- 卡片用于表格、表单、详情、设置、分类树等真实模块。
- 不要卡片套卡片。
- 不要每个小模块都独立大卡片。
- 卡片间距统一，不能出现因为网格行高导致的大空白。
- 边框不要过多，详情页流程区尤其避免多层边框。

## 文字

- 页面标题清晰。
- 辅助说明只在需要时保留，重复说明要去掉。
- 表格、表单、详情内字体大小适中，不能太小。
- 按钮文字不能溢出。

## 颜色

- 主色使用当前平台蓝色。
- 次级按钮使用灰色底。
- 危险操作使用红色。
- 状态颜色必须稳定：处理中蓝色、成功绿色、警告橙色、异常红色、禁用灰色。

## 顶部栏与面包屑

- 顶部栏参考后台系统：面包屑在左，搜索/通知/用户在右。
- 面包屑不包含多余系统名。
- 一级页面可只显示 `工作台` 或当前栏目。
- 二级页面显示父级和当前页，例如 `工作台 / 数据列表 / 详情页`。

## 表格

- 表格标题行要明确。
- 页面级操作按钮在表格卡片标题右侧。
- 表头和单元格对齐。
- 时间列排序图标靠近列名。
- 默认排序上下都不选中。
- 表格行头像保持正圆。

## 筛选区

- 筛选项左对齐。
- 搜索/重置与筛选项保持一致节奏。
- 不要把搜索/重置按钮推得太远。
- 筛选区不显示重复标题和重复重置入口。

## 表单

- 表单主体居中或在内容区内有稳定最大宽度。
- 标题、说明、字段在同一卡片内。
- 底部按钮右对齐。
- 上传使用灰色按钮。

## 详情

- 详情摘要高度克制。
- 状态圆点 + 文案。
- 优先级紧凑标签。
- 流程使用步骤条或流程节点。
- 标题头部无返回箭头时，内部左对齐。
