# 视觉与布局规则

本文件记录用户偏好的本地后台视觉规范。生成原型时以这些规则为准。

## 范围

- 只做 PC 端后台 / B 端 / Admin 原型。
- 不做移动端。
- 不做暗黑模式。
- 不做营销页、落地页、宣传页。
- 第一屏必须是实际可操作后台界面。

## 布局

- 真实后台系统优先：左侧导航、顶部栏、内容区。
- 不把所有模块堆在一个页面上。
- 页面通过导航、按钮、表格行、弹窗、抽屉形成真实工作流。
- 页面内容保持工作台密度，避免过度装饰。

## 卡片

- 卡片用于表格、表单、详情、设置、分类树等真实模块。
- 不要卡片套卡片。
- 不要每个小模块都独立大卡片。
- 卡片间距统一，不能出现因为网格行高导致的大空白。
- 边框不要过多，详情页流程区尤其避免多层边框。

## 文字

- 页面标题清晰。
- 辅助说明只在需要时保留，重复说明要去掉。
- 表格、表单、详情内字体大小适中，不能太小。
- 按钮文字不能溢出。

## 超长业务文本

- 所有超长业务文本遵循 `long-text-display.md`，该文件是完整规则的唯一详细定义。
- 先按用户当前任务选择完整展示、三行展开/收起或紧凑省略，再应用对应视觉表现。
- 数据层始终保留完整原始值；长文本不得挤压头像、图标、状态和操作区，也不得通过固定字符数切割字符串。

## 颜色

- 主色使用当前平台蓝色。
- 次级按钮使用灰色底。
- 危险操作使用红色。
- 状态颜色必须稳定：处理中蓝色、成功绿色、警告橙色、异常红色、禁用灰色。

## 顶部栏与面包屑

- 顶部栏参考后台系统：面包屑在左，搜索/通知/用户在右。
- 面包屑不包含多余系统名。
- 一级页面可只显示 `工作台` 或当前栏目。
- 二级页面显示父级和当前页，例如 `工作台 / 数据列表 / 详情页`。

## 表格

- 表格标题行要明确。
- 页面级操作按钮在表格卡片标题右侧。
- 表头和单元格对齐。
- 时间列排序图标靠近列名。
- 默认排序上下都不选中。
- 表格行头像保持正圆。

## 筛选区

- 筛选项左对齐。
- 搜索/重置与筛选项保持一致节奏。
- 不要把搜索/重置按钮推得太远。
- 筛选区不显示重复标题和重复重置入口。

## 表单

- 表单主体居中或在内容区内有稳定最大宽度。
- 标题、说明、字段在同一卡片内。
- 底部按钮右对齐。
- 上传使用灰色按钮。
- 必填项必须使用表单组件的 `required` 能力生成红色星号；禁止把 `*` 直接写进 label 文本。
- 每一个使用 `required` 标记的字段都必须实现真实校验，不能只显示红色星号或仅禁用提交按钮。用户点击提交/保存/确认后，空值或不合法值必须在对应字段附近显示红色错误状态和具体说明。
- 必填校验至少有两层触发：字段失焦后校验当前字段；点击提交/保存/确认后校验全部必填字段。校验失败时必须阻止提交并保留当前输入；用户修正字段后，该字段的错误状态应立即消失。错误文案使用可执行的“请输入…”“请选择…”或明确的长度/格式要求，禁止静默返回。
- 弹窗/抽屉的“保存”“确认”必须把必填校验绑定到保存前置守卫或关闭前置守卫（如 `before-ok` / `before-submit`），校验失败时不得执行保存处理器、不得修改列表/详情数据、不得关闭弹窗、不得显示成功结果。仅在 `ok` 事件里 `return` 不算合格。
- Select / Dropdown 触发器必须保持单行、正常控件高度，文本与后缀箭头垂直居中且不能换行。
- 禁止用未限定范围的父级 `span { display: ... }`、`svg { ... }` 等规则覆盖 Select / Dropdown 内部结构；文本样式只作用于明确的业务文案容器。
- 所有可填写或可选择控件都必须有初始展示：有明确业务默认值时显示真实默认值；没有默认值时必须提供清晰 placeholder，不能出现无文字的空白控件。
- placeholder 按控件类型写动作说明：输入框/数字框/文本域使用“请输入…”，下拉框/级联/树选择使用“请选择…”，日期时间使用“请选择…日期/时间”，搜索框说明可搜索的字段范围。
- 默认值与 placeholder 语义和样式必须区分：默认值代表已选中或已填写的数据，使用正常主文本色；placeholder 只是填写提示，使用弱提示色。不得用 placeholder 冒充默认值，也不得把示例值预填成真实业务数据。
- 编辑场景优先回填已有真实值；新增场景仅在 PRD 明确规定默认值时预选，否则展示 placeholder。

## 详情

- 详情摘要高度克制。
- 状态圆点 + 文案。
- 优先级紧凑标签。
- 流程使用步骤条或流程节点。
- 标题头部无返回箭头时，内部左对齐。
- 详情内容区的相邻模块必须保持统一纵向间距，默认 `16px`；状态流转、工单信息、时间线等模块不能上下贴合。
